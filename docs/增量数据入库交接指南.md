# 标准管理系统：增量数据入库交接指南 (SOP)

## 🎯 导言：致数据录入专员
这份文档是你日常工作的唯一准则。你**不需要**懂数据库表结构（什么 `std_base`、`unit_dict`），也**不需要**懂复杂的 Python 代码。
你只需要做好两件事：**填好统一的 Excel 表格**，**敲一行命令**。系统会自动完成清洗、查重、拆分、关联和入库。

---

## 🛠️ 第一部分：定义“大一统”数据源模板
为了实现真正的“傻瓜式”交接，绝不允许再出现“一个来源写一个脚本”的情况。以后所有新来的数据，必须让你（或业务人员）将其整理成**统一表头**的 Excel 文件。

### 必填核心列（表头名称必须一字不差）
| 表头名称 | 填写说明 | 示例 |
| :--- | :--- | :--- |
| **标准号** | 最核心字段，系统靠它查重（必填） | `GB/T 1234-2024` |
| **标准类别** | 决定数据去向，限填：`国标/行标/地标/团标/国际` | `地标` |
| **中文名称** | 标准全称 | `信息技术服务规范` |
| **状态** | 限填：`现行/废止/即将实施` | `现行` |

### 选填业务列（没有数据的列可以留空，但表头需保留）
| 表头名称 | 适用类别 | 填写说明 |
| :--- | :--- | :--- |
| **英文名称** | 通用 | 有则填，系统会自动在句末补 `.` |
| **发布日期** | 通用 | 推荐格式 `YYYY-MM-DD` |
| **实施日期** | 通用 | 推荐格式 `YYYY-MM-DD` |
| **起草单位** | 通用 | 多个单位用逗号 `,` 或顿号 `、` 隔开 |
| **替代标准** | 通用 | 填老标准号（如 `GB 123-2000`） |
| **备案号** | 行标/地标 | 业务专有字段 |
| **行业分类** | 行标/地标 | 业务分类 |
| **国民经济分类** | 团标 | 填 GBC 编码 |
| **提出单位** | 地标 | 业务专有字段 |
| **批准单位** | 地标 | 业务专有字段 |

## 🚀 第二部分：操作执行流程

### 步骤 1：准备数据
将整理好的 Excel 文件放入专用的待处理文件夹。
> **良好习惯**：文件命名写清楚批次，如 `202605_新增环保地标500条.xlsx`。

### 步骤 2：执行“一键入库”脚本
打开服务器的命令行界面（终端），输入以下命令执行大一统录入脚本：

```bash
python E:\建表\auto_import_master.py --file "C:\待处理\202605_新增环保地标500条.xlsx"
```

**底层防呆机制（无需你操心）：**
1. **自动防重**：如果 `标准号` 库里已经有了，系统自动跳过，**绝对不会覆盖**老数据。
2. **自动清洗**：单位名称里的 `(盖章)`、`(有限责任公司)` 会被自动剥离。
3. **自动分流**：系统看到 `地标` 就会往 `std_db_detail` 写，看到 `国标` 就会往 `std_gb_detail` 写。

### 步骤 3：挂载 PDF 附件文件（文件入库）
如果这批标准还附带了 PDF 源文件：
1. **拷文件**：将 PDF 全选，直接拖进 NAS 对应的网盘分类文件夹里（如 `Y:\磁盘阵列\标准文件下载\地标下载\`）。
2. **跑关联**：不用去数据库一条条改路径，直接运行自动化扫盘脚本：
```bash
python E:\建表\scan_and_import_filepath.py
```
*(脚本会自动从 PDF 文件名中提取标准号，去数据库寻找并建立绑定映射。)*

## ⚠️ 第三部分：异常排查指南 (Troubleshooting)

**情况 1：脚本刚跑就报错退出了？**
* **排查**：打开 Excel，检查第一行的表头是不是多加了空格，或者少填了必填列（如把“标准类别”写成了“类别”）。脚本是严格按列名读取的。

**情况 2：单位信息进不去，或者关联错乱？**
* **排查**：Excel 里的单位之间是不是用了换行符（Alt+Enter）或者奇怪的特殊符号隔开？请用查找替换功能，统一改成英文逗号 `,`。

**情况 3：扫盘脚本跑完后，有些 PDF 在系统里点不开？**
* **排查**：说明 PDF 文件名太乱，系统没认出来。
* **解决**：打开 `E:\建表\scan_filepath_log.txt`，翻到底部看 `[UNMATCHED]` 列表。如果是文件命名不规范（如 `新建文档(1).pdf`），你只需要把文件重命名为规范的 `DB11/T 123-2024.pdf`，然后重新跑一次步骤 3 即可。

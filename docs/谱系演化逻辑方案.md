# 标准管理系统：谱系演化逻辑重构方案

您提出的这个问题非常敏锐，直接触及了数据结构的核心痛点！

## 🎯 痛点诊断：为什么 `{}` 会丢失关系？
目前的 `{A} 、 {B、C} 、 {D、E}` 写法，在计算机算法中叫做**“广度优先遍历（BFS）的层级输出”**。
它的致命缺陷是：**只记录了“辈分”，丢掉了“血缘”**。
它只能告诉我们 D 和 E 都是“爷爷辈”，但无法记录到底 D 是 B 的亲生父亲，还是 C 的亲生父亲。如果用这种数据给前端画图（您发的图3），前端根本画不出那些连线（Edge），只能画出一排排孤立的圆圈。

为了彻底解决这个问题，我为您设计了以下两套方案（强烈推荐方案一）。

---

## 🏆 方案一：算法全自动推导（彻底解放录入员，强烈推荐！）

**核心理念：让操作员只管“眼前”，让算法去追溯“祖宗十八代”。**

真实业务中，一个标准发布时，只会宣布“本标准替代了某某老标准”，绝对不会在封面上把七八十年代的老标准全列出来。所以，让操作员在 Excel 里手动去查、去拼接长长的 `{}` 链条，既不合理，又极易出错。

### 1. Excel 录入端的改变
操作员在 Excel 的 `替代标准` 列，**只填写直接被替代的老标准**。
*   例如，A 发布了，替代了 B 和 C，Excel 里就填：`B, C`。
*   绝对不要去管 B 和 C 以前又替代了谁。

### 2. 数据库底座：`std_replace` 就是完美的图
当 Web 工具把 `A -> B` 和 `A -> C` 写进 `std_replace` 表时，这张表其实已经构成了一个完美的**有向无环图 (DAG)**。

### 3. 当场计算生成 `ped_chain`（您疑惑 3 的终极答案）
当新标准 A 入库后，我们在代码里写一个**递归算法（深度优先搜索 DFS）**：
1. 程序发现 A 替代了 B 和 C。
2. 程序自动去 `std_replace` 表里查：B 替代过谁？查到 B 替代了 D。
3. 程序接着查：C 替代过谁？查到 C 替代了 E。
4. **程序自动在后台为您组装出完整的谱系图，并写入 `std_pedigree` 和 `std_ped_chain`。**

### 4. `ped_chain` 存储格式升级（为了前端能画出图3）
既然要给前端画那种漂亮的拓扑树（图3），就必须抛弃 `{}` 文本，改用 **JSON 邻接表**存储。
*   **新版 `ped_chain` 存储内容：**
    ```json
    {
      "nodes": ["A", "B", "C", "D", "E"],
      "edges": [
        {"source": "A", "target": "B"},
        {"source": "A", "target": "C"},
        {"source": "B", "target": "D"},
        {"source": "C", "target": "E"}
      ]
    }
    ```
有了这串 JSON，您的前端团队（无论是用 ECharts, AntV 还是 D3.js），两行代码就能直接渲染出图 3 的关系树，一根线都不会连错！

## 🛠️ 方案二：Excel 纯手工连线方案（妥协做法）

如果您目前的业务场景，**必须**要求操作员在 Excel 里把整棵树的演化史一次性全敲进去，那就必须改变录入语法，改用**“边集合 (Edge List)”**语法。

### Excel 录入语法规则
放弃按“代”分组，改为按“动作”分组。使用 `=>` 代表替代，使用 `|` 分隔不同动作。

*   **输入示例**：`A=>[B,C] | B=>[D] | C=>[E]`
*   **语法解释**：
    *   `A=>[B,C]` 明确表示 A 取代了 B 和 C。
    *   `B=>[D]` 明确表示 B 取代了 D。
    *   这种写法，不论树有多复杂、交叉多少次，在逻辑上绝对不会产生歧义。

### 脚本解析逻辑
Web 录入工具在读到这串文本时，按 `|` 切割，按 `=>` 提取父子关系，同样可以精准还原树状图，并将其转为 JSON 写入 `std_ped_chain` 表。

## 💡 架构师建议
您发的图 2 里的 `ped_chain`（ `{GB 1000.1-1988} 、 {GB 1000-1981}` 这种格式），是**上一个时代**遗留下来的非结构化数据，它只适合人类肉眼看，极度不适合计算机画图和计算。

**强烈建议采用【方案一】**：
1. 老数据的 `{}` 文本留着做历史兼容。
2. 新工具入库时，直接按**方案一**跑递归，把树状结构生成标准的 JSON 存进新字段（或者覆盖老字段）。
3. 这不仅解决了录入分不清上下级的问题，更直接把前端画图（图3）的底层数据结构给铺平了！

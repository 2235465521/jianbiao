# 标准管理系统 V3：全库入库规则手册（完整版）

本手册旨在指导后续新增标准数据的标准化入库。系统采用“核心表 + 详情分表 + 维度表”的架构，所有入库操作必须严格遵循以下规则。

---

## 1. 核心信息层 (Core Data)

### 1.1 `std_base` (基础信息表)
**作用**：产生标准唯一的数字 ID (`base_id`)。
- **入库规则**：
    - `std_id`: 标准号（如 `GB/T 1-2020`），必须唯一且去除首尾空格。
    - `std_type_no`: `00` 国标, `01` 行标, `02` 地标, `03` 团标。
    - `std_englishname`: 若由程序翻译，末尾必须补 `.` 符号。
- **操作逻辑**：首先插入该表，获取自增 ID。
- **SQL 示例**：
  ```sql
  INSERT INTO std_base (std_id, std_chinesename, std_englishname, std_status, std_type_no) 
  VALUES ('GB/T 1234-2024', '示例标准', 'Example Standard.', '现行', '00');
  ```

### 1.2 `std_xx_detail` (详情分表)
**作用**：根据标准类型，存储特定业务字段。
- **入库规则**：
    - `base_id`: 必须关联 `std_base` 的 ID。
    - `gbc`: 仅限团标 (`std_tb_detail`)，国民经济分类编码。
    - `regi_no`: 仅限行标 (`std_hb_detail`)，备案号字段。
- **操作逻辑**：根据 `std_type_no` 分流进入 4 个详情表之一。

---

## 2. 单位与地域画像层 (Unit & Geo)

### 2.1 `unit_dict` (单位实体字典)
**作用**：存储标准化的单位名称及行政归属。
- **清洗规则**：
    - 去除冗余括号，提取内部地域名（如“福州”）。
    - 过滤掉长度 <= 3 的“人名”或脏数据。
- **操作逻辑**：先查 `unit_name` 是否存在，若无则插入获取 `unit_id`。

### 2.2 `std_unit_relation` (标准-单位映射表)
**作用**：建立标准与起草单位的多对多关系。
- **入库规则**：
    - `role_type`: `1` (第一起草人), `2` (参与起草人)。
    - `rank_order`: 起草单位出现的先后顺序（1, 2, 3...）。

### 2.3 `area_dict` (行政区划字典表)
**作用**：系统地理统计的基础。
- **字段规范**：包含 `area_code` (6位), `level` (1-3), `longitude`, `latitude`。
- **维护建议**：通常保持静态，仅在行政区划调整时更新。

---

## 3. 关系与演进层 (Evolution)

### 3.1 `std_replace` (替代关系表)
**作用**：记录标准的新旧更替。
- **入库规则**：
    - `base_id`: 新标准的数字 ID，查不到则填 `NULL`。
    - `replace_id`: 被替代标准的数字 ID，查不到则填 `NULL`。
    - `replace_std_name`: **必须填写**原始字符串编号（如 `GB 123-1980`）。
    - `replace_type`: `0` 部分替代, `1` 全部替代, `2` 部分代完。

### 3.2 `std_pedigree` & `std_ped_chain` (谱系表)
**作用**：追踪标准的全生命周期演化链。
- **规范**：`ped_id` 支持长字符串；`ped_chain` 使用 ` > ` 符号连接演化路径。

---

## 4. 索引与辅助层 (Auxiliary)

### 4.1 `std_index` (全文检索索引表)
**作用**：加速全局搜索。
- **规范**：`index_context` 存储 JSON 格式的检索词。

### 4.2 `std_extend_h` & `std_extend_s` (历史遗留表)
- **状态**：**已停止维护**。新数据统一走 `unit_dict` 体系。

---

## 🚀 总结：新数据入库标准 6 步走
1. **清理数据**：清洗标准号、翻译缺失名。
2. **基础落库**：插入 `std_base` 并拿回自增 ID。
3. **详情落库**：根据分类进 `detail` 对应的表。
4. **单位落库**：清洗单位名 -> 查/增 `unit_dict` -> 写 `std_unit_relation`。
5. **关系落库**：读取替代信息，转换 ID 后写入 `std_replace`。
6. **刷新索引**：同步更新 `std_index` 的 JSON 内容。
